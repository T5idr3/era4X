#領地探索整理の下準備 2017/04/17

	##内容
		呼び出し方法については様々な構想があるのではと思い、
		どんな処理でもとりあえず必要そうで、量が多く手間がかかりそうな下準備について
		済ませることを念頭に作業してみました。

		関数名その他、全てご自由に改変してください。
		製作者様、領地探索製作者の皆様に感謝。

	##詳細
		+SELECTCASE文からイベント出して1CASEを1関数に変更しました。
		+イベント終了時にRETURN 1を返すようにしました。
		+イベントを別ファイルにまとめました。
		+コメントアウトされていたイベントは別ファイルへ移動。連番を外しました。
		+コメントアウトされていたイベントは発生するイベントと同じ形にして、コメントアウトしなおしました。
		+残った発生するイベントのCASE番号を連番で振り直しました。それでも222種類で大量。
		+各関数に必要なDIM宣言を追加しました。
		+【調教SLG】eratoho総合スレ【売却金額229】>>636 >>781【調教SLG】eratoho総合スレ【好感度230】>>824辺り
			ご注意のあった{}や@"、気づいたところを修正しました。
		+本体からTRYCCALLFORMで呼び出すようにしました。
			RETURN 0の場合は関数が見つからなかった場合と同じ扱いにしました。

	##DIM宣言追加について備考
		###「金貸しと農民」の仲間の評価が上がる処理について
			+以下の処理について、変数「仲間」の選出処理が存在しなかったため、DIM宣言を追加しませんでした。
				IF 仲間 != 0 && 対象 == MASTER
					CFLAG:仲間:好感度 += 100
					CALL COLORPRINT(@"%ANAME(仲間)%の評価が上がった", カラー_注意, "W")
				ENDIF

			+候補キャラの選出を追加しようかと思ったのですが……。
				文脈から仲間全員の好感度を上昇させたかったのではと思い、勝手ながら以下にしました。
				意図が違った場合は直して頂けると幸いです。
				FOR LOCAL:0, 0, CHARANUM
					IF CFLAG:(LOCAL:0):所属 == CFLAG:対象:所属 && !CFLAG:(LOCAL:0):捕虜先 && !IS_ANIMAL(LOCAL:0)
						CFLAG:(LOCAL:0):好感度 += 100
					ENDIF
				NEXT
				CALL COLORPRINT(@"%ANAME(対象)%に対する仲間の評価が上がった", カラー_注意, "W")
